Bridge Engine SDK Reference

Welcome, Bridge Engine Developer!

The Bridge Engine SDK is made of the Bridge Engine Framework, sample applications to get you started, and a reference framework documentation generated from its headers.

You can find out more about Bridge Engine app development at: http://bridge.occipital.com/developers/.

Important note: the BridgeEngine framework is not compatible with the iOS simulator, you will need to plug in a physical iOS device to deploy and test your app.

Framework Overview

The Bridge Engine framework defines an Objective-C interface for developing Mixed Reality applications leveraging the Structure Sensor and the Bridge Headset on iOS devices.

#import <BridgeEngine/BridgeEngine.h>

Sample Apps (Native)

The following sample apps have no non-standard dependencies (other than the Bridge Engine framework), and can be built and run with Xcode alone.

MixedReality

The MixedReality Sample application is a simple application that will let you place some virtual objects in the scene.

Bridget

The Bridget Sample application showcases the capabilities of Bridget, our cute little robot. Bridget can interact with the user, navigate around obstacles, and play fetch!

Unity Plugin

One Unity package is available to make it easy to get started with Bridge Engine for developers who are familiar with Unity. The package uses Bridge Engine SDK, and can be extended as needed by developers.

See the Unity folder for more, including Unity Plugin Overview.

App Development

Developing applications with Bridge Engine requires Xcode 8 or above, freely available on the Apple developer site.

External Accessory Protocols

Important: When creating a new application using the Bridge Engine SDK, the main Info.plist needs to contain an additional key "Supported external accessory protocols", with the following array of values:

  • io.structure.control
  • io.structure.depth
  • io.structure.infrared
Without this modification to the property list, the application will not be able to connect to the sensor. All sample applications have this key/value array. Check them out!

iOS Frameworks

The Bridge Engine framework depends on the following iOS frameworks:

  • Accelerate.framework
  • AVFoundation.framework
  • CoreImage.framework
  • CoreGraphics.framework
  • CoreMedia.framework
  • CoreMotion.framework
  • CoreVideo.framework
  • ExternalAccessory.framework
  • Foundation.framework
  • ImageIO.framework
  • Metal.framework
  • OpenGLES.framework
  • QuartzCore.framework
  • UIKit.framework
  • libz.dylib

You will therefore need to link them into your app too.
